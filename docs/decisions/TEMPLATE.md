# ADR Template

Copy this entry to the top of `DECISIONS.md` when making an architectural decision.

```
## YYYY-MM-DD — [Short title]

**Decision**: [What was decided, in one sentence]
**Why**: [The rationale — what problem does this solve?]
**Alternatives considered**: [What was rejected and why]
**Impact**: [Optional — what modules/files are affected]
```

## When to log a decision

- Adding a new abstraction layer or pattern
- Choosing between two valid approaches
- Adding or removing a dependency
- Changing security/safety behavior
- Modifying data models or storage
- Introducing a new convention

## When NOT to log

- Bug fixes with obvious solutions
- Formatting/style changes
- Test additions that follow existing patterns
