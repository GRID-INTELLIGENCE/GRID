## How to report

Please report any security issue by emailing security@grid.dev (preferred) or by opening a confidential/limited-access issue if your GitHub organization / repository supports private security reports.

Include:
- A clear description of the issue and the components or file paths affected
- Step-by-step instructions to reproduce the issue (a minimal PoC is very helpful)
- Any relevant logs, stack traces, or environment details
- Your contact information so we can follow up if necessary

If you'd like to share a vulnerability privately and securely, please encrypt your report using the project's PGP key (if published) and attach it in the email.

## Response & timeline

We aim to:
- Acknowledge receipt within 48 hours
- Provide an initial assessment and potential remediation plan within 7 days
- Work with contributors for any fix and coordinate a public disclosure timeline

If a fix requires coordinated disclosure, we will not publish details until a release containing the fix is available or until an agreed-upon disclosure date.
